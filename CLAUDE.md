# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

`legged_lab` is an Isaac Lab extension for legged-robot reinforcement learning. It implements **DeepMimic** and **Adversarial Motion Priors (AMP)** for humanoid robots (primarily Unitree G1). The RL backend is a forked `rsl_rl` library (branch `feature/amp`) that must be installed separately and placed at `../rsl_rl/` relative to this repo.

## Running Python Commands

**Never run Python commands directly.** Always use the `legged-lab-runner` skill for `python`, `pytest`, `pip`, and `pre-commit`.

## Key Commands

**Training:**
```bash
python scripts/rsl_rl/train.py --task LeggedLab-Isaac-AMP-G1-v0 --headless --max_iterations 50000
python scripts/rsl_rl/train.py --task LeggedLab-Isaac-Deepmimic-G1-v0 --headless --max_iterations 50000
# Multi-GPU:
python scripts/rsl_rl/train.py --task LeggedLab-Isaac-AMP-G1-v0 --headless --device cuda:1 agent.device=cuda:1
```

**Playback:**
```bash
python scripts/rsl_rl/play.py --task LeggedLab-Isaac-AMP-G1-v0 --headless --num_envs 64 --video --checkpoint logs/rsl_rl/<exp>/<run>/model_xxx.pt
```

**Tests:**
```bash
pytest source/legged_lab/test/
pytest source/legged_lab/test/test_codex_env.py  # single test file
```

**Linting / formatting:**
```bash
pre-commit run --all-files   # black (120-char), isort, flake8, pyupgrade, codespell
```

**Motion data retargeting:**
```bash
python scripts/tools/retarget/dataset_retarget.py \
    --robot g1 --input_dir temp/gmr_data/ --output_dir temp/lab_data/ \
    --config_file scripts/tools/retarget/config/g1_29dof.yaml
```

**List registered envs:**
```bash
python scripts/list_envs.py
```

## Architecture

```
source/legged_lab/legged_lab/
├── envs/                   # Custom env classes (extend Isaac Lab ManagerBasedRLEnv)
│   ├── manager_based_animation_env.py   # Base: adds AnimationManager + MotionDataManager
│   └── manager_based_amp_env.py         # AMP: adds terminal obs preview for disc observations
├── managers/               # Custom Isaac Lab managers
│   ├── motion_data_manager.py   # Loads .pkl motion clips; samples reference frames per env
│   ├── animation_manager.py     # Plays reference animation alongside the robot
│   └── preview_observation_manager.py  # Captures obs before reset for AMP discriminator
├── tasks/locomotion/
│   ├── amp/                # AMP task: env cfg + mdp (rewards, observations, symmetry)
│   ├── deepmimic/          # DeepMimic task: env cfg + mdp
│   ├── animation/          # Animation playback task
│   └── velocity/           # Baseline velocity-tracking task (Go2, G1)
├── assets/                 # Robot articulation configs (Unitree G1, Go2, actuators)
├── rsl_rl/                 # Runner/algorithm config dataclasses for rsl_rl integration
├── sensors/                # Custom sensor configs (ray caster array)
└── utils/                  # Math helpers (quat slerp, ang vel from quat diff, etc.)
```

**Task registration** follows Isaac Lab conventions: `__init__.py` files in each `config/<robot>/` directory register gym environments via `gym.register(...)`.

**Config pattern**: All env configs use Isaac Lab's `@configclass` decorator. Robot-specific configs live under `tasks/locomotion/<task>/config/<robot>/` and inherit from the base `<Task>EnvCfg`. Agent (PPO/AMP runner) configs live in `config/<robot>/agents/`.

**AMP data flow**: `MotionDataManager` loads `.npz` motion clips → `AnimationManager` steps the reference ghost robot → `PreviewObservationManager` snapshots discriminator observations before env reset → `ManagerBasedAmpEnv.step()` merges the preview into `extras["terminal_obs"]` for the AMP runner.

**Motion data format**: `.npz` files contain: `fps`, `root_pos`, `root_rot` (wxyz), `dof_pos`, `body_names`, `body_pos_w`, `body_quat_w`, `body_lin_vel_w`, `body_ang_vel_w`. All positions are relative to `env_origin`. Generated by `scripts/tools/retarget/` from GMR output. Key bodies for AMP/DeepMimic are configured via `MotionDataTermCfg.key_body_names` and indexed at runtime from `body_pos_w`.

## Docker Workflow

Host layout expected by default:
```
.../lab_dev/
├── legged_lab/     # this repo
└── rsl_rl/         # forked rsl_rl (feature/amp branch)
```

```bash
bash docker/build.sh          # build image
# xhost +
bash docker/run.sh            # start container (mounts source, installs packages)
bash docker/enter.sh          # attach shell (working dir: /workspace/legged_lab)
bash docker/stop.sh           # stop & remove container
```

Override `RSL_RL_PATH` in `docker/.env.base` if `rsl_rl` is not at the default location.

## Coding Style

- Python 3.11, 4-space indent, 120-char line limit
- Formatter: `black --preview`; import sorter: `isort` with Black profile
- `legged_lab` is `known_firstparty`; Isaac Lab packages are `known_isaaclabparty`
- Task/config filenames: `<robot>_<task>_env_cfg.py` (e.g. `g1_amp_env_cfg.py`)
- `__init__.py` in tasks directories is only for gym registration; keep it minimal

## Assets & Large Files

Large assets (USD robot models, `.npz` motion data) are tracked via **Git LFS**. Run `git lfs pull` after cloning or when assets are updated. Do not commit files from `logs/`, `outputs/`, or `temp/`.

## Git

- Check current working directory before every git operation (main repo vs. this subrepo).
- Always run `pre-commit` before committing.
- Show proposed commit scope and message to the user and wait for confirmation before committing.
- Use English for all commit messages.
